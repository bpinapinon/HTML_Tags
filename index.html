Header2: Anchor Tags

<a> tags are for creating hyperlinks in webpages. At a minimum, we need to provide an "href" attribute to our anchor tag. We pass the URL we want out hyperlink to direct to into this "href" attribute. This serves a similar purpose as the "src" attribute does for images.

For example, this HTML:
<a href="https://github.com/">Link to Github</a>

produces this hyperlink:
Link to GitHub

Learn More about Anchor Tags

Header2: Unordered and Ordered Lists

<ul> and <li> tags are for rendering an unordered list in your HTML document.

For example, this HTML:
<ul>
  <li>Monday</li>
  <li>Tuesday</li>
  <li>Thursday</li>
</ul>

produces this list in our document:

  • Monday
  • Tuesday
  • Thursday

We can produce an ordered list simply by swapping out the ul element for an ol element.
<ol>
  <li>Monday</li>
  <li>Tuesday</li>
  <li>Thursday</li>
</ol>

produces this list in our document:

  1. Monday
  2. Tuesday
  3. Thursday

Header2: Tables

<table> tags are for representing tabular data inside of a table. On their own, table tags don't do much. So we nest <tr> (row of data), <td> (populate row with data), <th> (headers) tags.

For example, this HTML:

<table border="1">
  <tr>
    <th>Company</th>
    <th>Contact</th>
    <th>Email</th>
  </tr>
  <tr>
    <td>Google</td>
    <td>Sergey Brin</td>
    <td>sergey@google.com</td>
  </tr>
  <tr>
    <td>Facebook</td>
    <td>Mark Zuckerburg</td>
    <td>mark@facebook.com</td>
  </tr>
   <tr>
    <td>Space X</td>
    <td>Elon Musk</td>
    <td>elon@spacex.com</td>
  </tr>
  <tr>
    <td>Amazon</td>
    <td>Jeff Bezos</td>
    <td>jeff@amazon.com</td>
  </tr>
</table>

Produces this table:

Company Contact Email
Google Sergey Brin sergey@google.com
Facebook Mark Zuckerburg mark@facebook.com
Space X Elon Musk elon@spacex.com
Amazon Jeff Bezos jeff@amazon.com

Header2: Division Tags

<div> tags are meant to be generic container elements with no semantic meaning other than those given to them by their classes and id attributes.
You can group elements together by throwing them inside of the same div tag.

There are other more semantic HTML tags such as articles or navs or footers which behave the exact same way as div elements do, but are designed to be used for specific purposes. These should be used whenever possible for a few reasons, but one of the most practical is that code is easier to understand and debug when things are named what they actually are.
